# cosam-modify: CLI Editing Tool

## Description

`cosam-modify` provides command-line access to the schedule edit system for scripted
or batch modifications. It supports all entity types via the field system, with all
changes recorded in the CRDT (automerge) document. Input can be native binary or xlsx;
output is always native binary.

### CLI syntax (v9-style multi-stage)

```text
cosam-modify --file <path> [--new] [--format text|json|toml]
             [--select <type> [<query>]] <command> [args]
             [-- [--select <type> [<query>]] <command> [args] ...]
```

### Commands

- `list` — display all or selected entities with their field values
- `get <query>` — display a single entity
- `set <field> <value>` — update a field on selected entities
- `create <field>=<value> ...` — create a new entity
- `delete <query>` — soft-delete an entity
- `add-edge <edge-field> <target>` — add a relationship
- `remove-edge <edge-field> <target>` — remove a relationship
- `undo` / `redo` — reverse or re-apply the last edit (in-memory)
- `show-history` — show undo/redo stack depth

### Not in scope

- xlsx write output (until IDEA-080 and related are implemented)
