# UI-087: Hide event-type filter chips for types with no panels

## Summary

The Event Type filter shows all non-hidden panel types even when none of that
type appear in the loaded schedule.

## Status

Completed

## Priority

High

## Description

Panel types such as "To Be Announced", "Fan Demonstration", "Meeting",
"Meet the Guest", and "Room Hours" appear as filter chips even when zero
panels of those types are present in the schedule data. Selecting one of
these chips immediately shows "No events match your filters," which is
confusing.

## Implementation

The fix belongs in the exporter (`schedule-core`), not the widget: only
panel types that are actually referenced by at least one panel should be
included in the `panelTypes` map in the output JSON.

## Acceptance Criteria

- The exported JSON `panelTypes` object contains only types referenced by
  at least one panel in the `panels` array.
- Break, hidden, timeline, private, and other special types continue to be
  included only if they have panels referencing them.
