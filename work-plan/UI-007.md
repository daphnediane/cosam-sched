# Improve room filter to exclude room-hours-only rooms

## Summary

Update the room filter dropdown to only include rooms that have scheduled panels, excluding rooms that only contain room-hours events (RH prefix or "Is Room Hours" flag).

## Status

Open

## Priority

Medium

## Description

Currently the room filter shows all rooms from the Rooms sheet, including rooms that only contain room-hours events like "Market Expo" or "Registration". These rooms clutter the filter and don't contain actual panels that users want to filter by.

## Expected behavior

- Room filter should only show rooms with actual panel events
- Rooms with only room-hours events should be excluded from filter
- Rooms with both room-hours and panel events should be included
- Filter should be more useful and less cluttered

## Implementation Details

### Room Classification Logic

1. **Distinguish room types during conversion**:

   ```perl
   sub _classify_room_events ($events, $room_id) {
       my $has_panels = 0;
       my $has_room_hours = 0;
       
       for my $event (@$events) {
           next unless $event->{roomId} == $room_id;
           
           if ($event->{is_room_hours}) {
               $has_room_hours = 1;
           } elsif (!$event->{is_break}) {
               $has_panels = 1;
           }
       }
       
       return {
           has_panels => $has_panels,
           has_room_hours => $has_room_hours,
           filterable => $has_panels
       };
   }
   ```

2. **Update JSON room data**:

   ```json

         "uid": 1,
         "name": "Panel Room 1",
         "long_name": "Main Panel Room",

       {

   ```

### Widget Filter Updates

1. **Update room filter building**:

   ```javascript
   function buildRoomFilter(rooms) {
       return rooms
           .filter(room => room.filterable)
           .sort((a, b) => a.sort_key - b.sort_key);
   }
   ```

2. **Filter dropdown population**:
   - Only include rooms with `filterable: true`
   - Maintain existing sort order
   - Keep same filter behavior for included rooms

### Conversion Process Updates

1. **Enhance room processing**:

   ```perl
   sub classify_room_events {
       my ($events) = @_;
       my %room_stats;
       
       for my $event (@$events) {
           my $room_id = $event->{roomId};
           next unless $room_id;
           
           $room_stats{$room_id}{has_panels} = 1
               unless $event->{is_room_hours};
           $room_stats{$room_id}{has_room_hours} = 1
               if $event->{is_room_hours};
       }
       
       return \%room_stats;
   }
   ```

2. **Backward compatibility**:

   - Default to `filterable: true` for existing data
   - Graceful handling if classification data missing
   - Maintain current filter behavior for rooms without classification

### Testing

   - Rooms with both panels and room-hours (should be filterable)
   - Empty rooms (should not be filterable)
   - Legacy data without classification (default to filterable)
