# Switch from tables to proper grid layout

## Summary

Replace table-based layout with CSS grid similar to schedule-to-html implementation.

## Status

Open

## Priority

High

## Description

The current implementation uses HTML tables for layout, which is inflexible and doesn't provide the modern responsive design capabilities needed. The schedule-to-html system uses CSS grid which provides better control over layout, responsive behavior, and visual presentation.

## Implementation Details

- Study the grid layout implementation in schedule-to-html/output/flyers/byday/Friday.html
- Replace table structures with CSS grid containers
- Implement responsive grid columns for different screen sizes
- Ensure proper grid cell spanning for multi-time events
- Maintain all existing functionality during the transition
- Test grid layout across different devices and screen sizes

## Acceptance Criteria

- Schedule displays using CSS grid instead of tables
- Responsive behavior works on mobile and desktop
- Multi-time events span grid cells correctly
- All existing filters and features continue to work
- Performance is maintained or improved
