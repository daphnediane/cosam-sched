# Visual conflict indicators in schedule widget

## Summary

Add visual indicators to the schedule widget to highlight conflicting events, making it easy for users to identify and understand scheduling conflicts.

## Status

Open

## Priority

Medium

## Description

The converter now includes conflict data in the JSON output, but the widget doesn't display this information to users. Users need visual cues to quickly identify conflicting events and understand what the conflicts are.

Based on the 2025 schedule data, conflicts include:

* Room conflicts (Market Expo vs Learn to solder sessions)
* Presenter conflicts (UNC Staff double-booked)

## Expected behavior

* Conflicting events should have visual indicators (warning icons, colors, etc.)
* Hovering/clicking should show conflict details
* Different visual treatments for different conflict types
* Clear distinction between real conflicts and allowed overlaps (room-wide events)

## Implementation Details

### Visual Indicators

1. **Conflict icons**:
   * Add warning icon (⚠️) to conflicting event cards
   * Use different colors for different conflict types:
     * Red border for room conflicts
     * Orange border for presenter conflicts
     * Yellow border for group presenter conflicts (if shown)

2. **Event styling modifications**:
   * Add CSS classes for conflict states:

     ```css
     .cosam-grid-event.conflict-room { border-left: 4px solid #dc3545; }
     .cosam-grid-event.conflict-presenter { border-left: 4px solid #fd7e14; }
     .cosam-grid-event.conflict-group { border-left: 4px solid #ffc107; }
     ```

3. **Conflict badge/overlay**:
   * Small badge showing conflict count
   * Position in top-right corner of event card
   * Color-coded by conflict type

### Interaction Design

1. **Hover tooltip**:
   * Show conflict details on hover
   * List conflicting events with names and times
   * Include conflict type and resolution suggestions

2. **Click to view conflicts**:
   * Click on conflicting event to highlight related events
   * Dim non-conflicting events
   * Show conflict details panel

3. **Conflict summary panel**:
   * Optional panel showing all conflicts in current view
   * Filterable by conflict type
   * Links to jump to conflicting events

### Widget JavaScript Updates

1. **Event rendering enhancement**:

   ```javascript
   function renderEventWithConflicts(event) {
       const eventEl = createEventElement(event);
       
       if (event.conflicts && event.conflicts.length > 0) {
           eventEl.classList.add('has-conflicts');
           
           // Add conflict-specific classes
           event.conflicts.forEach(conflict => {
               eventEl.classList.add(`conflict-${conflict.type}`);
           });
           
           // Add conflict indicator
           const indicator = createConflictIndicator(event.conflicts);
           eventEl.appendChild(indicator);
       }
       
       return eventEl;
   }
   ```

2. **Conflict tooltip system**:

   ```javascript
   function createConflictTooltip(conflicts) {
       const tooltip = document.createElement('div');
       tooltip.className = 'conflict-tooltip';
       
       conflicts.forEach(conflict => {
           const item = document.createElement('div');
           item.className = `conflict-item conflict-${conflict.type}`;
           item.textContent = conflict.details;
           tooltip.appendChild(item);
       });
       
       return tooltip;
   }
   ```

3. **Conflict highlighting**:

   ```javascript
   function highlightConflictingEvents(eventId) {
       // Find event and its conflicts
       const event = findEvent(eventId);
       const conflictIds = event.conflicts.map(c => c.conflict_event_id);
       
       // Highlight conflicting events
       document.querySelectorAll('.cosam-grid-event').forEach(el => {
           const elId = el.dataset.eventId;
           if (elId === eventId || conflictIds.includes(elId)) {
               el.classList.add('conflict-highlighted');
           } else {
               el.classList.add('conflict-dimmed');
           }
       });
   }
   ```
