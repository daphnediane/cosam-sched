# Work Plan Tracking Rules

## Work Plan Structure

Work items are stored as individual markdown files in the `work-plan/` directory with the format: `<PREFIX>-<###>.md`

### Prefixes (Categories)

- **FEATURE** - New functionality or major enhancements
- **BUGFIX** - Fixes for defects or issues
- **UI** - User interface improvements and visual changes
- **PERFORMANCE** - Optimizations and performance improvements
- **DOCS** - Documentation updates
- **REFACTOR** - Code restructuring without functional changes
- **TEST** - Test additions or improvements

### Work Item File Layout

Each work item file must follow this structure:

```markdown
# Brief title

## Summary
One-line summary of the work item.

## Status
Open | In Progress | Completed | Blocked

## Priority
High | Medium | Low

## Description
Detailed description of the work to be done.

## Additional Sections
- Steps to Fix (for bugs)
- Implementation Details (for features)
- Acceptance Criteria
- Notes
```

### Numbering Rules

- Keep the `-###` suffix unique across all work items
- When moving an item between categories, preserve the original number
- New items should use the next available number regardless of prefix
- Check existing files to avoid number collisions

## Work Plan Management

### Adding New Work Items

1. Create a new file in `work-plan/` with appropriate prefix and next available number
2. Follow the standard file layout
3. Set initial status to "Open"
4. Regenerate `WORK_PLAN.md` using: `perl work-plan/combine_workplans.pl`

### Updating Work Items

- Edit the individual work item file directly
- Update status as work progresses (Open → In Progress → Completed)
- Regenerate `WORK_PLAN.md` after changes

### Regenerating WORK_PLAN.md

The combined work plan is generated by running:

```bash
# From project root
perl work-plan/combine_workplans.pl

# Or using the shell wrapper
./work-plan/update_workplan.sh
```

This creates `WORK_PLAN.md` in the project root with:

- Items organized by priority (High, Medium, Low)
- Completed items show only summary
- Open items show full details with links to individual files

## Markdown Formatting

All work plan files must follow markdown lint rules:

- Blank lines before and after headings (MD022)
- Blank lines before and after lists (MD032)
- Use the formatting script to fix issues: `perl work-plan/fix_markdown_format.pl`
