# Work Plan Tracking Rules

## Work Plan Structure

Work items are stored as individual markdown files in the `docs/work-plan/` directory with the format: `<PREFIX>-<###>.md`

### Prefixes (Categories)

- **FEATURE** - New functionality or major enhancements
- **BUGFIX** - Fixes for defects or issues
- **UI** - User interface improvements and visual changes
- **PERFORMANCE** - Optimizations and performance improvements
- **DOCS** - Documentation updates
- **CLEANUP** - Repository structure and technical debt cleanup
- **REFACTOR** - Code restructuring without functional changes
- **TEST** - Test additions or improvements
- **EDITOR** - Desktop schedule editor application (Rust/GPUI)

### Work Item File Layout

Each work item file must follow this structure:

```markdown
# Brief title

## Summary
One-line summary of the work item.

## Status
Open | In Progress | Completed | Blocked

## Priority
High | Medium | Low

## Description
Detailed description of the work to be done.

## Additional Sections
- Steps to Fix (for bugs)
- Implementation Details (for features)
- Acceptance Criteria
- Notes
```

### Numbering Rules

- Keep the `-###` suffix unique across all work items
- When moving an item between categories, preserve the original number
- New items should use the next available number regardless of prefix
- Check existing files to avoid number collisions

## Work Plan Management

### Adding New Work Items

1. Create a new file in `docs/work-plan/` with appropriate prefix and next available number
2. Follow the standard file layout
3. Set initial status to "Open"
4. Regenerate `docs/WORK_PLAN.md` using: `perl scripts/combine-workplans.pl`

### Updating Work Items

- Edit the individual work item file directly
- Update status as work progresses (Open → In Progress → Completed)
- Regenerate `docs/WORK_PLAN.md` after changes

### Regenerating docs/WORK_PLAN.md

The combined work plan is generated by running:

**On Unix-like systems (Linux/macOS):**

```bash
perl ./scripts/combine-workplans.pl
```

**On Windows:**

```powershell
.\scripts\combine-workplans.ps1
```

Both scripts automatically resolve paths relative to their location, so they can be run from any directory.

This creates `docs/WORK_PLAN.md` with:

- Items organized by priority (High, Medium, Low)
- Completed items show only summary
- Open items show full details with links to individual files

## Markdown Formatting

All work plan files must follow markdown lint rules.

**Recommended Approach:**
Use the following IDE extensions for automatic markdown formatting:

1. **Install Extensions:**
   - `DavidAnson.vscode-markdownlint` - The actual markdownlint formatter
   - `franneck94.workspace-formatter` - Directory-level formatting tool

2. **Format Directory:**
   - Right-click on the `docs/work-plan/` directory
   - Select "Format Directory" from the context menu
   - The extensions will automatically fix all markdown formatting issues

**Alternative Manual Approach:**
Open individual files and save them to trigger the IDE's built-in markdownlint auto-formatter.
