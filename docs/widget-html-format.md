# Widget HTML Display Format

This document describes the `widget-html` embedded format used by the CosAm calendar
widget when schedule data is embedded as semantic HTML. This format is distinct from the
pure JSON format described in `widget-json-format.md`, but shares the same structural
data schema for all sections except panels.

## Purpose and Goals

The widget-html format exists to provide two capabilities simultaneously:

- **SEO and crawlability**: Panel names, descriptions, times, rooms, and credits are
  rendered as real HTML text nodes, readable by search engines and screen readers without
  JavaScript execution.
- **Progressive enhancement**: The HTML list is a fully readable schedule without
  JavaScript. When JavaScript is available, the widget replaces it with the full
  interactive UI (day tabs, grid/list views, filtering, search, starring, QR sharing).

## Top-Level Structure

Inside the widget root element (`#cosam-calendar-root` or any configured root), the
widget-html format consists of two parts:

1. A compact JSON block carrying all structural/reference data **except panels**.
2. A semantic HTML section containing one element per panel.

```html
<div id="cosam-calendar-root">
  <!-- Part 1: Structural metadata (no panels) -->
  <script type="application/json" data-cosam="schedule">
    { "meta": {…}, "rooms": […], "panelTypes": {…}, "timeline": […], "presenters": […] }
  </script>

  <!-- Part 2: Semantic panel list -->
  <section class="cosam-static-schedule" aria-label="Schedule">
    <article class="cosam-panel" data-id="GP001" …>…</article>
    …
  </section>
</div>
```

The widget detects this format automatically when it finds a
`script[data-cosam="schedule"]` element inside the root element and no separate
`dataEl` or `data` option is provided.

## Part 1: JSON Block

### Detection

```html
<script type="application/json" data-cosam="schedule">
  { … }
</script>
```

The `<script type="application/json">` element is not executed by browsers (the MIME
type is not a JavaScript type), is invisible to users, and is safe to embed inside any
block-level element.

### Schema

The JSON object has the same schema as the widget JSON display format (see
`widget-json-format.md`), with two differences:

- The `panels` field is **absent** — panels are encoded as HTML elements instead.
- `meta.variant` is `"html-embedded"` instead of `"display"`.

```json
{
  "meta": {
    "title": "Event Schedule 2026",
    "version": 0,
    "variant": "html-embedded",
    "generator": "cosam-convert 0.1.0",
    "generated": "2026-03-26T22:00:00Z",
    "modified": "2026-03-26T21:45:00Z",
    "startTime": "2026-05-29T17:00:00",
    "endTime": "2026-06-01T15:00:00"
  },
  "rooms": [ … ],
  "panelTypes": { … },
  "timeline": [ … ],
  "presenters": [ … ]
}
```

The `rooms`, `panelTypes`, `timeline`, and `presenters` arrays/objects are identical
to their counterparts in the widget JSON format. See `widget-json-format.md` for their
full schemas.

## Part 2: Panel HTML Elements

### Container

```html
<section class="cosam-static-schedule" aria-label="Schedule">
  …panel articles…
</section>
```

Panel articles are ordered to match the widget JSON panels array: scheduled panels
before unscheduled, then within scheduled panels by earliest `startTime` → longest
duration → lowest room UID → `id` → `name`.

Break panels generated by the exporter (panel type `%IB`, `%NB`) are included in
the HTML elements and identified by their `data-panel-type` attribute. In the
static (no-JS) view, these may be visually suppressed via CSS targeting
`[data-panel-type="%IB"]` and `[data-panel-type="%NB"]` if desired.

### Panel Element Schema

Each panel is an `<article class="cosam-panel">` element.

#### Required `data-*` Attributes

| Attribute         | Type    | Description                                                    |
| ----------------- | ------- | -------------------------------------------------------------- |
| `data-id`         | string  | Full session ID (e.g. `"GP001"`, `"GW097P1S2"`)                |
| `data-base-id`    | string  | Base panel ID (e.g. `"GP001"`, `"GW097"`)                      |
| `data-panel-type` | string  | Panel type prefix; references `panelTypes` hash key            |
| `data-room-ids`   | string  | Space-separated room UIDs (e.g. `"1 3"`); empty string if none |
| `data-start-time` | string  | ISO 8601 local datetime (no timezone suffix)                   |
| `data-end-time`   | string  | ISO 8601 local datetime (no timezone suffix)                   |
| `data-duration`   | integer | Duration in minutes                                            |
| `data-is-premium` | boolean | `"true"` or `"false"`                                          |
| `data-is-full`    | boolean | `"true"` or `"false"`                                          |
| `data-is-kids`    | boolean | `"true"` or `"false"`                                          |

#### Optional `data-*` Attributes

| Attribute             | Type    | Description                                                                     |
| --------------------- | ------- | ------------------------------------------------------------------------------- |
| `data-part-num`       | integer | Part number (omitted when absent)                                               |
| `data-session-num`    | integer | Session number (omitted when absent)                                            |
| `data-total-parts`    | integer | Distinct part count for a multi-part series (omitted unless >1 part)            |
| `data-is-series-lead` | boolean | `"true"` on the lead part that bears the shared series cost (omitted otherwise) |
| `data-cost`           | string  | Cost string (see Cost Values documentation)                                     |
| `data-capacity`       | string  | Seat capacity (e.g. `"15"`)                                                     |
| `data-difficulty`     | string  | Skill level indicator                                                           |
| `data-ticket-url`     | string  | Ticket purchase URL                                                             |

#### Required Child Elements

| Selector                | Content                                                     |
| ----------------------- | ----------------------------------------------------------- |
| `h3.cosam-panel-name`   | Panel name (text content)                                   |
| `time.cosam-panel-time` | Human-readable time range; `datetime` attr = ISO start time |
| `p.cosam-panel-rooms`   | Room display name(s), comma-separated if multiple           |

#### Optional Child Elements

| Selector                 | Content                                        |
| ------------------------ | ---------------------------------------------- |
| `p.cosam-panel-desc`     | Description text (omitted if empty)            |
| `p.cosam-panel-note`     | Note text (omitted if empty)                   |
| `p.cosam-panel-prereq`   | Prerequisite text (omitted if empty)           |
| `ul.cosam-panel-credits` | Formatted credit strings; one `<li>` per entry |

The `credits` array is reconstructed by the widget as
`Array.from(el.querySelectorAll('.cosam-panel-credits > li')).map(li => li.textContent)`.

### Example Panel Element

```html
<article class="cosam-panel"
  data-id="GP001"
  data-base-id="GP001"
  data-panel-type="GP"
  data-room-ids="10"
  data-start-time="2026-06-26T14:00:00"
  data-end-time="2026-06-26T15:00:00"
  data-duration="60"
  data-is-premium="false"
  data-is-full="false"
  data-is-kids="false"
>
  <header>
    <h3 class="cosam-panel-name">Example Panel Title</h3>
    <time class="cosam-panel-time" datetime="2026-06-26T14:00:00">
      Friday 2:00 PM – 3:00 PM
    </time>
    <p class="cosam-panel-rooms">Workshop Room 1</p>
  </header>
  <p class="cosam-panel-desc">Example panel description.</p>
  <ul class="cosam-panel-credits">
    <li>Presenter Name</li>
    <li>Group Name (Alice, Bob)</li>
  </ul>
</article>
```

## Widget Initialization

### Auto-Detection

When `CosAmCalendar.init()` is called with only an `el` option (no `dataEl`, `data`,
or `dataUrl`), the widget checks for the presence of a `script[data-cosam="schedule"]`
element inside the root element. If found, it uses the widget-html parsing path.

```js
CosAmCalendar.init({
  el: document.getElementById('cosam-calendar-root'),
});
```

### Data Reconstruction

The widget reconstructs the full data model from the HTML as follows:

1. **Structural data**: Parse the `script[data-cosam="schedule"]` JSON → provides
   `meta`, `rooms`, `panelTypes`, `timeline`, `presenters`.

2. **Panels array**: Query all `article.cosam-panel` elements in DOM order (already
   chronological). For each element:
   - Read all `data-*` attributes for scalar fields.
   - Read `.cosam-panel-name`, `.cosam-panel-desc`, `.cosam-panel-note`,
     `.cosam-panel-prereq` text content for string fields.
   - Read `.cosam-panel-credits > li` text content list for the `credits` array.

3. **`panel.presenters`** (individual names for search and filtering): The `presenters`
   field on each panel object is **not stored in the HTML**. Instead, it is reconstructed
   from the JSON `presenters[].panelIds` data: for each presenter entry, for each panel
   ID in `panelIds`, add the presenter's name to that panel's `presenters` list.

The resulting data object is structurally equivalent to the widget JSON format and is
passed through the same `_normalizeDataModel()` transformation as other data sources.

## Format Comparison

| Aspect              | widget JSON (`display`)            | widget-html (`html-embedded`)               |
| ------------------- | ---------------------------------- | ------------------------------------------- |
| Panel data location | JSON `panels` array (compressed)   | Semantic HTML `<article>` elements          |
| Structural data     | Inline in JSON                     | `<script data-cosam="schedule">` JSON block |
| SEO / crawlable     | No (base64 blob)                   | Yes (plain HTML text)                       |
| No-JS fallback      | "Please enable JavaScript" message | Readable schedule list                      |
| Page size           | Smaller (gzip compressed)          | Larger (uncompressed HTML)                  |
| Widget features     | Full                               | Full (same data model after parsing)        |

## Generating the Format

The widget-html format is produced by `cosam-convert` using the `--export-embed` flag
(the default embed output). The Rust generator is in
`apps/cosam-convert/src/static_html.rs`.

The generator:

1. Serializes the JSON block with `meta.variant = "html-embedded"` and no `panels` field.
2. Iterates `export.panels` sorted by `startTime`.
3. For each panel, writes an `<article class="cosam-panel">` with all required `data-*`
   attributes and child elements.
4. Looks up room display names from `export.rooms` for the `p.cosam-panel-rooms` element.
