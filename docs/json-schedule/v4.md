# Schedule JSON Format v4

This document describes version 4 of the schedule JSON format. V4 introduces timeline support and time types while maintaining backward compatibility with earlier versions.

## Top-Level Structure

```json
{
  "meta": { ... },
  "timeline": [ ... ],
  "events": [ ... ],
  "rooms": [ ... ],
  "panelTypes": [ ... ],
  "timeTypes": [ ... ],
  "presenters": [ ... ],
  "conflicts": [ ... ]
}
```

All top-level keys are required except `conflicts`, which may be omitted when there are no scheduling conflicts.

## Structures

- [meta](meta-v4.md) - Metadata about the schedule file
- [timeline](timeline-v4.md) - Key time markers for layout and navigation
- [events](events-v4.md) - Array of scheduled events
- [rooms](rooms-v4.md) - Physical and virtual event spaces
- [panelTypes](panelTypes-v4.md) - Event category definitions
- [timeTypes](timeTypes-v4.md) - Time category definitions
- [presenters](presenters-v4.md) - People and groups that present events
- [conflicts](conflicts-v4.md) - Detected scheduling conflicts

## Key Changes from v3

- **Added `timeline` array**: Key time markers used for layout, navigation, and formatting
- **Added `timeTypes` array**: Time category definitions (replaces split panel types)
- **Added `meta.startTime` and `meta.endTime`**: Schedule bounds calculation
- **Removed `events[].kind` field**: Now derived from `panelTypes[].kind`
- **Removed `panelTypes[].isSplit` field**: Split types moved to timeTypes

## Version History

- **v1** (implicit): Original format with no `version` field
- **v2**: Added `meta.version` and `meta.generator` fields
- **v3**: Added `panelTypes[].isSplit` field, removed event-level workshop/break flags
- **v4**: Added timeline and timeTypes, removed event.kind, added schedule bounds

## Migration Notes

### v3 → v4

1. Convert panel types with `isSplit: true` to timeTypes entries
2. Create timeline entries for split events
3. Add `meta.startTime` and `meta.endTime` from schedule bounds
4. Remove `events[].kind` field (derive from panel type)
5. Remove `panelTypes[].isSplit` field

### Backward Compatibility

V4 files can be read by V3 parsers with the following considerations:

- New fields (`timeline`, `timeTypes`, `meta.startTime/endTime`) will be ignored
- Missing `events[].kind` will need to be derived from panel type if needed
- Timeline functionality will not be available
