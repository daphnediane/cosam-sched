# Schedule JSON Format v5 - Public/Widget

This document describes version 5 of the schedule JSON format, public/widget variant. This format is produced by the Rust converter or editor in public export mode and consumed by the schedule widget.

## Top-Level Structure

```json
{
  "meta": { ... },
  "panels": [ ... ],
  "rooms": [ ... ],
  "panelTypes": [ ... ],
  "timeTypes": [ ... ],
  "timeline": [ ... ],
  "presenters": [ ... ]
}
```

`conflicts` is omitted from public output. `panels` is a **flat ordered array** pre-flattened from the private hierarchical format.

## Structures

- [meta](meta-v5.md) - Metadata about the schedule file (shared with private format)
- [panels](panels-public-v5.md) - Flattened panels array with pre-computed effective values
- [rooms](rooms-v4.md) - Physical and virtual event spaces (same as v4)
- [panelTypes](panelTypes-v4.md) - Event category definitions (same as v4)
- [timeTypes](timeTypes-v4.md) - Time category definitions (same as v4)
- [timeline](timeline-v4.md) - Key time markers for layout and navigation (same as v4)
- [presenters](presenters-v4.md) - People and groups that present events (same as v4)

## Key Changes from v4

- **Flattened panels array**: Replaces `events` array with session-level entries
- **Pre-computed effective values**: All hierarchy levels resolved for simple rendering
- **Public-only fields**: Private/internal fields filtered out
- **Enhanced credits**: Formatted credit strings with group resolution
- **Variant support**: `meta.variant` distinguishes public from private format
- **No conflicts array**: Conflict detection is internal-only

## Panel Entry Structure

Each entry in the `panels` array represents one **session** - the smallest schedulable unit:

### Identification Fields

- `id`: Full Uniq ID (e.g., `"GW097P1S2"`, `"GP002"`)
- `baseId`: Base panel ID (e.g., `"GW097"`, `"GP002"`)
- `partNum`: Part number or `null`
- `sessionNum`: Session number or `null`

### Content Fields

- `name`: Display name from base panel
- `description`: Effective description (concatenated across hierarchy)
- `note`: Effective note
- `prereq`: Effective prerequisite text

### Scheduling Fields

- `roomIds`: Room UIDs for this session
- `startTime`/`endTime`: ISO 8601 local times
- `duration`: Duration in minutes

### Display Fields

- `panelType`: Panel type UID
- `cost`: Cost string from base
- `capacity`: Effective seat capacity
- `difficulty`: Skill level from base
- `ticketUrl`: Effective ticket URL
- `isFree`/`isFull`/`isKids`: Boolean flags

### Credits and Presenters

- `credits`: Formatted credit strings for public display
- `presenters`: All individual presenter names for filtering

## Array Ordering

Entries are ordered chronologically by `startTime`. Unscheduled sessions (null `startTime`) appear at the end.

## Effective Value Computation

All values that accumulate across hierarchy levels are pre-computed by the exporter:

### Concatenated Fields

`description`, `note`, and `prereq` are joined with spaces across base→part→session levels.

### Override Fields

`ticketUrl` uses first non-null value (session → base).

### Credits Generation

1. If `hidePanelist` is true, `credits` is empty
2. If `altPanelist` exists, credits contains that single string
3. Otherwise, credits are generated by group resolution from all credited presenters

### Presenter List

`presenters` contains all individual presenter names (credited + uncredited) for filtering.

## Migration Notes

### v4 → v5-public

1. Convert `events` array to flattened `panels` array
2. Add hierarchy fields (`baseId`, `partNum`, `sessionNum`)
3. Pre-compute effective values across hierarchy levels
4. Generate formatted `credits` array
5. Add `presenters` array for filtering
6. Set `meta.variant = "public"`
7. Remove `conflicts` array
8. Preserve all existing v4 structures (rooms, panelTypes, etc.)

### Private → Public Conversion

1. Flatten hierarchical `panels` hash to ordered array
2. Compute effective values for each session
3. Generate credits using group resolution
4. Filter out private/internal fields
5. Add presenter lists for filtering
6. Order chronologically by start time

## Example

```json
{
  "meta": {
    "title": "Cosplay America 2026 Schedule",
    "generated": "2026-06-01T12:00:00Z",
    "version": 5,
    "variant": "public",
    "generator": "cosam-editor 0.2.0",
    "startTime": "2026-06-26T17:00:00Z",
    "endTime": "2026-06-28T18:00:00Z"
  },
  "panels": [
    {
      "id": "GP002",
      "baseId": "GP002",
      "partNum": null,
      "sessionNum": null,
      "name": "Cosplay Contest Misconceptions",
      "panelType": "panel-type-gp",
      "roomIds": [10],
      "startTime": "2026-06-26T14:00:00",
      "endTime": "2026-06-26T15:00:00",
      "duration": 60,
      "description": "A deep-dive into competition issues.",
      "note": null,
      "prereq": null,
      "cost": null,
      "capacity": null,
      "difficulty": null,
      "ticketUrl": null,
      "isFree": true,

      "isKids": false,

  "rooms": [],
  "panelTypes": [],
  "timeTypes": [],
  "timeline": [],
  "presenters": []
}
```

## Notes

- This format is optimized for simple rendering in the schedule widget
- No hierarchy understanding required - all values are pre-computed
- Private/internal fields are filtered out for public consumption
- The widget only needs to display the flattened session entries
- Credits are formatted for display while presenters are used for filtering
