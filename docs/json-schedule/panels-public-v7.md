# Panels Structure v7 - Public Format

`panels` is a JSON array where each entry represents one session - the smallest schedulable unit, flattened from the private hierarchical format with enhanced support for implicit breaks.

**Access:** Public

**Status:** Supported in v7 (public format only)

## Key Changes from v6

- **panelType Format**: Now uses 2-letter prefix (e.g., "BR") instead of full UID ("panel-type-br")
- **Implicit Breaks**: Automatically generated break sessions are included in the array
- **BREAK Room**: Special room for implicit breaks with sort_key in hidden range

## Fields

| Field         | Type            | Public | Description                                                  |
| ------------- | --------------- | ------ | ------------------------------------------------------------ |
| `id`          | string          | yes    | Full Uniq ID of this session (e.g. `"IB001S1"`, `"GP002"`)   |
| `baseId`      | string          | yes    | Base panel ID (e.g. `"IB001"`, `"GP002"`)                    |
| `partNum`     | integer \| null | yes    | Part number; `null` when no part subdivision                 |
| `sessionNum`  | integer \| null | yes    | Session number; `null` when no session subdivision           |
| `name`        | string          | yes    | Display name (from base panel)                               |
| `panelType`   | string \| null  | yes    | Panel type prefix (e.g. `"BR"`, `"IB"`, `"NB"`)               |
| `roomIds`     | integer[]       | yes    | Room UIDs for this session; empty array if unscheduled       |
| `startTime`   | string \| null  | yes    | ISO 8601 local datetime; null if unscheduled                 |
| `endTime`     | string \| null  | yes    | ISO 8601 local datetime                                      |
| `duration`    | integer         | yes    | Duration in minutes                                          |
| `description` | string \| null  | yes    | Effective description (base + part + session concatenated)   |
| `note`        | string \| null  | yes    | Effective note                                               |
| `prereq`      | string \| null  | yes    | Effective prerequisite text                                  |
| `cost`        | string \| null  | yes    | Cost string from base (see Cost Values in v4 documentation)  |
| `capacity`    | string \| null  | yes    | Effective seat capacity (session override or base default)   |
| `difficulty`  | string \| null  | yes    | Skill level indicator (from base)                            |
| `ticketUrl`   | string \| null  | yes    | Effective ticket URL (session override or base default)      |
| `isFree`      | boolean         | yes    | True if no additional cost                                   |
| `isFull`      | boolean         | yes    | True if this session is at capacity                          |
| `isKids`      | boolean         | yes    | True for kids-only panels                                    |
| `credits`     | string[]        | yes    | Formatted credit strings for public display                  |

## Implicit Break Examples

### Standard Implicit Break

```json
{
  "id": "IB001S1",
  "baseId": "IB001", 
  "partNum": null,
  "sessionNum": null,
  "name": "Break",
  "panelType": "IB",

  "startTime": "2026-06-26T15:00:00",

```

## BREAK Room

Implicit breaks are assigned to a special "BREAK" room:

```json
{
  "uid": 999,
  "short_name": "BREAK",
  "long_name": "BREAK", 
  "hotel_room": "",
  "sort_key": 999
}
```

The BREAK room uses a high sort_key (999) to place it in the hidden range while still being accessible to the widget.

## Auto-Generation Logic

Implicit breaks are generated by the backend when:

1. **Gap Detection**: A gap of 3+ hours exists between events in all non-hidden rooms
2. **No Conflicts**: No explicit breaks are scheduled during the gap period  
3. **Room Coverage**: The gap affects all non-hidden rooms simultaneously
4. **Time Calculation**: Duration calculated from previous event end to next event start

### Overnight Detection

A break is marked as overnight when:
- Events span different calendar days, OR
- The time period crosses 4:00 AM boundary

### ID Generation

- **Standard Breaks**: `IB001`, `IB002`, etc. (Implicit Break)
- **Overnight Breaks**: `NB001`, `NB002`, etc. (Overnight Break)  
- **Session IDs**: `{baseId}S1` (single session per break panel)

## Widget Integration

The widget handles v7 format by:

1. **Prefix Conversion**: Converting `panelType` prefixes to CSS classes (`"IB"` → `panel-type-ib`)
2. **Break Detection**: Using `panelType` to identify breaks (no longer needs client-side generation)
3. **Room Display**: Showing BREAK room for implicit breaks
4. **Backward Compatibility**: Supporting both v6 (full UIDs) and v7 (prefixes) formats

## Migration from v6

- **panelType**: `"panel-type-br"` → `"BR"`
- **Implicit Breaks**: No longer generated client-side
- **CSS Classes**: Widget handles prefix-to-UID conversion automatically
