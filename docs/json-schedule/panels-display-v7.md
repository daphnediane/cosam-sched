# `panels` (Display Format)

`panels` is a JSON array where each entry represents one **session** - the smallest schedulable unit, flattened from the full hierarchical format. In v7, the display variant also includes baked-in implicit break panels.

## Access

Public

## Status

Supported in v7 (display format only)

## Fields

| Field         | Type            | Public | Description                                                     |
| ------------- | --------------- | ------ | --------------------------------------------------------------- |
| `id`          | string          | yes    | Full Uniq ID of this session (e.g. `"GW097P1S2"`, `"GP002"`)    |
| `baseId`      | string          | yes    | Base panel ID (e.g. `"GW097"`, `"GP002"`)                       |
| `partNum`     | integer \| null | yes    | Part number; `null` when no part subdivision                    |
| `sessionNum`  | integer \| null | yes    | Session number; `null` when no session subdivision              |
| `name`        | string          | yes    | Display name (from base panel)                                  |
| `panelType`   | string \| null  | yes    | Panel type prefix (e.g. `"GW"`), references panelTypes hash key |
| `roomIds`     | integer[]       | yes    | Room UIDs for this session; empty array if unscheduled          |
| `startTime`   | string \| null  | yes    | ISO 8601 local datetime; null if unscheduled                    |
| `endTime`     | string \| null  | yes    | ISO 8601 local datetime                                         |
| `duration`    | integer         | yes    | Duration in minutes                                             |
| `description` | string \| null  | yes    | Effective description (base + part + session concatenated)      |
| `note`        | string \| null  | yes    | Effective note                                                  |
| `prereq`      | string \| null  | yes    | Effective prerequisite text                                     |
| `cost`        | string \| null  | yes    | Cost string from base (see Cost Values in v4 documentation)     |
| `capacity`    | string \| null  | yes    | Effective seat capacity (session override or base default)      |
| `difficulty`  | string \| null  | yes    | Skill level indicator (from base)                               |
| `ticketUrl`   | string \| null  | yes    | Effective ticket URL (session override or base default)         |
| `isFree`      | boolean         | yes    | True if no additional cost                                      |
| `isFull`      | boolean         | yes    | True if this session is at capacity                             |
| `isKids`      | boolean         | yes    | True for kids-only panels                                       |
| `credits`     | string[]        | yes    | Formatted credit strings for public display                     |
| `presenters`  | string[]        | yes    | All individual presenter names for filtering and search         |

## Description

The display panels array is a flattened representation of the hierarchical full format, optimized for simple rendering in the schedule widget. Each entry represents a single session with all effective values pre-computed.

### Key Differences from v5 Public Format

- **`panelType`** uses prefix directly (e.g. `"GP"`) instead of `"panel-type-gp"` UID format
- **Baked-in breaks**: Implicit breaks are included as regular panel entries with synthetic panel types (`%IB`, `%NB`)
- **No metadata**: The `metadata` field is stripped from all items in the display variant
- **Variant name**: `"display"` instead of `"public"`

### Baked-In Breaks

The display variant includes implicit break panels generated by the exporter:

- **`%IB` (Implicit Break)**: Generated for gaps between scheduled panels during active hours. Break panels are expanded to cover all inactive visible rooms during their time slot.
- **`%NB` (Overnight Break)**: Generated for overnight gaps between programming days.

The widget no longer needs to compute implicit breaks at runtime.

### Array Ordering

Entries are ordered chronologically by `startTime`. Unscheduled sessions (null `startTime`) appear at the end.

### Effective Values

All fields that accumulate across hierarchy levels are pre-computed:

- **Concatenated fields**: `description`, `note`, `prereq` are joined with spaces
- **Override fields**: `ticketUrl` uses first non-null value (session → base)
- **Capacity**: Session override or base default
- **Presenters**: Union of all presenters across base, part, and session levels

### Credits Generation

The `credits` array is computed by the exporter:

1. If `hidePanelist` is true on the session, `credits` is an empty array
2. Otherwise, if an effective `altPanelist` value exists, credits contains that single string
3. Otherwise, credits are generated by group resolution from the union of all `creditedPresenters`

## Examples

```json
[
  {
    "id": "GP002",
    "baseId": "GP002",
    "name": "Cosplay Contest Misconceptions",
    "panelType": "GP",
    "roomIds": [10],
    "startTime": "2026-06-26T14:00:00",
    "endTime": "2026-06-26T15:00:00",
    "duration": 60,
    "description": "A deep-dive into competition issues.",
    "isFree": true,
    "credits": ["December Wynn", "Pros and Cons Cosplay"],
    "presenters": ["December Wynn", "Pro", "Con"]
  },
  {
    "id": "%IB001",
    "baseId": "%IB001",
    "name": "Break",
    "panelType": "%IB",
    "roomIds": [1, 2, 3, 10],
    "startTime": "2026-06-26T15:00:00",
    "endTime": "2026-06-26T15:30:00",
    "duration": 30,
    "description": null,
    "isFree": true,
    "credits": [],
    "presenters": []
  }
]
```

## Notes

- This format is only used in the display (`"display"`) variant of v7
- Full format uses the hierarchical [panels hash](panels-v7.md) instead
- All effective values are pre-computed by the exporter for simple rendering
- The widget does not need to understand the base→part→session hierarchy
- `credits` contains formatted display strings, while `presenters` contains raw names for filtering
- Implicit break panels have synthetic panel type prefixes starting with `%`
- No `metadata` field is present on any item in the display variant
