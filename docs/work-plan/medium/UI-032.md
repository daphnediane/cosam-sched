# Implement sticky headers or additional header rows

## Summary

Add sticky headers or repeat day headers between time blocks in grid view for better navigation.

## Status

Open

## Priority

Medium

## Description

When viewing the schedule grid, users lose context of which day/time they're viewing as they scroll. Either sticky headers should follow the scroll, or additional header rows should be inserted between days to maintain context.

## Implementation Details

- Evaluate sticky header vs. repeated header approaches
- For sticky headers: implement CSS position:sticky with proper z-index management
- For repeated headers: insert day/time header rows at regular intervals in the grid
- Ensure headers don't interfere with event content
- Test scroll behavior on different devices and screen sizes
- Consider performance implications of sticky positioning

## Acceptance Criteria

- Users can always see current day/time context while scrolling
- Headers don't obscure event content
- Implementation works on mobile and desktop
- Performance remains acceptable
- Visual design is consistent with overall theme
