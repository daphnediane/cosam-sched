# CLI-094: cosam-modify create command

## Summary

Implement the `create` subcommand to add a new entity of any type with specified fields.

## Status

Open

## Priority

Low

## Blocked By

- CLI-092: cosam-modify list and get commands

## Description

`create` builds an `AddEntity` command with the user-supplied fields and applies it through
`EditContext::apply()`. The execute path calls the registered `build_fn` which runs
`build_entity` internally.

```text
cosam-modify --file <path> --select <type> create <field>=<value> [...]
```

or with named flags:

```text
cosam-modify --file <path> --select <type> create --field <name> <value> [...]
```

### UUID generation

`EntityId::<E>::generate()` — always generates a fresh v7 UUID (public, no schedule
needed). The `AddEntity` command is constructed with this UUID and the field list; the
execute path calls `build_fn` which inserts the entity via `build_entity` with
`UuidPreference::Exact(uuid)`. Do NOT pre-insert the entity.

### Field name resolution

Same as `set`: `E::field_set().get_by_name(name_str)` → `resolved.name()` for
`&'static str` field names in the `AddEntity` fields vector.
