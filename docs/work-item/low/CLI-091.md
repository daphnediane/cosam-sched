# CLI-091: cosam-modify scaffold, file I/O, module structure

## Summary

Establish the module layout, Cargo dependencies, arg-parsing skeleton, and file
load/save infrastructure for `cosam-modify`.

## Status

Open

## Priority

Low

## Blocked By

- CLI-090: schedule-core metadata update API

## Description

Creates the module skeleton for `cosam-modify` and wires the two core operations every
subcommand needs: loading a schedule (binary or xlsx) and saving it back as native binary.

### Module structure

```text
apps/cosam-modify/src/
  main.rs     — entry: parse args, load, run stages, save
  args.rs     — CliArgs / Stage / StageCommand types + parse_args()
  load.rs     — load_schedule() + save_schedule()
  output.rs   — OutputFormat (Text/Json/Toml) + formatting helpers
  cmd/mod.rs  — run_stage() dispatcher; submodules: list get set create delete edge undo
```

### CLI syntax (v9-style multi-stage)

```text
cosam-modify --file <path> [--new] [--format text|json|toml]
             [--select <type> [<query>]] <command> [args]
             [-- [--select <type> [<query>]] <command> [args] ...]
```

Stage separator is the literal `--` token. Arg parsing follows cosam-convert's manual
`while index < args.len()` pattern.

### File auto-detection

- Magic `COSAM\x00` → `Schedule::load_from_file()`
- Otherwise → `import_xlsx(path, &XlsxImportOptions::default())`
- `--new` or non-existent path → `Schedule::new()`
- Save always writes native binary (xlsx write out of scope until IDEA-080)

## Acceptance Criteria

- [ ] `cargo build -p cosam-modify` succeeds
- [ ] `cosam-modify --file /tmp/t.cosam --new list` runs without error
- [ ] Dirty flag is not set when no commands are run (no spurious saves)
