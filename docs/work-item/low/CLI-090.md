# CLI-090: schedule-core metadata update API for cosam-modify

## Summary

Add `Schedule::touch_modified()` and `EditContext::schedule_mut()` to schedule-core;
wire `touch_modified` into `apply()`, `undo()`, and `redo()`.

## Status

Open

## Priority

Low

## Blocked By

- CLI-031: cosam-modify: CLI editing tool (parent)

## Description

`EditContext::apply()` currently never updates `Schedule::metadata.modified_at`, so binary
files saved after edits always have a stale or missing modification timestamp.

### Changes

- `Schedule::touch_modified(&mut self)` in `schedule/crdt.rs` — sets
  `metadata.modified_at = Some(Utc::now())`. Does **not** touch `version` (schema version,
  always `0` until a schema migration requires a bump).
- `EditContext::schedule_mut(&mut self) -> &mut Schedule` in `edit/context.rs` — gives
  external callers (e.g. `cosam-modify`) mutable access to stamp `metadata.generator` before
  saving.
- Wire `self.schedule.touch_modified()` in `apply()`, `undo()`, and `redo()` after each
  successful execution.

## Acceptance Criteria

- [ ] `touch_modified()` sets `modified_at` to a non-None value
- [ ] `touch_modified()` does not change `version`
- [ ] `EditContext::apply()` calls `touch_modified()` after successful execute
- [ ] `EditContext::undo()` and `redo()` call `touch_modified()` after successful execute
- [ ] `cargo test -p schedule-core` passes with new tests covering the above
