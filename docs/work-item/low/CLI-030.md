# cosam-convert: Format Conversion Tool

## Summary

CLI tool for converting between schedule file formats (XLSX, native binary, widget JSON, HTML).

## Status

In Progress

## Priority

## Description

`cosam-convert` is a command-line application for importing and exporting
schedule data between supported formats. It uses a flag-based API with
settings chaining: output settings accumulate and are snapshotted into each
output job, enabling multiple outputs with different settings in one
invocation.

### Usage

```text
cosam-convert --input <file.xlsx|file.schedule> [options]
```

### Output commands

Multiple output commands may be specified; each captures the current settings:

- `--output, -o <file>` — save private schedule (`.xlsx` or native binary by extension)
- `--export, -e <file.json>` — export public widget JSON
- `--export-embed <file.html>` — export embeddable HTML (inline CSS/JS/JSON)
- `--export-test <file.html>` — export standalone test page (Squarespace simulator)

### Validation

- `--check` / `--validate` — report conflicts; exit non-zero if any found

### Table names (XLSX import)

- `--schedule-table <name>` (default: `Schedule`)
- `--roommap-table <name>` (default: `RoomMap`)
- `--prefix-table <name>` (default: `Prefix`)
- `--presenter-table <name>` (default: `Presenters`)

### Output settings (apply to subsequent outputs until overridden)

- `--title <string>` — event title for widget JSON and test pages
- `--widget <basename>` — set CSS and JS to `<basename>.css` and `<basename>.js`
- `--widget-css <path>` — override CSS source (default: builtin)
- `--widget-js <path>` — override JS source (default: builtin)
- `--test-template <path>` — override test page template (default: builtin)
- `--minified` (default) / `--no-minified` / `--for-debug`
- `--style-page` / `--no-style-page`
- `--builtin-css`, `--builtin-js`, `--builtin-widget`, `--builtin-template`, `--builtin`
- `--default` — reset all settings to defaults
