# CLI-092: cosam-modify list and get commands

## Summary

Implement the `list` and `get` subcommands to display entities and their field values.

## Status

Open

## Priority

Low

## Blocked By

- CLI-091: cosam-modify scaffold, file I/O, module structure

## Description

`list` displays all (or filtered) entities of a type. `get` displays a single entity
by name or UUID. Both use the field system to enumerate readable fields and edge fields,
producing output in `text`, `json`, or `toml` format.

### `list`

```text
cosam-modify --file <path> [--select <type> [<query>]] list
```

Entity types: `panel`, `presenter`, `event_room`/`room`, `hotel_room`, `panel_type`/`type`.
Default type: `panel`. No query / `*` / `all` → all entities; named query →
`lookup_list::<E>(schedule, query)`.

### `get`

```text
cosam-modify --file <path> [--select <type>] get <query>
```

Requires a non-empty query; uses `lookup_single::<E>()`.

### Field iteration

`E::field_set().readable_fields()` for plain fields plus `E::field_set().half_edges()` for
edge fields. Values read via `desc.read(id, schedule)`.

## Acceptance Criteria

- [ ] `list` with no query returns all entities
- [ ] `list` with a name query returns matching entities
- [ ] `get` with a UUID returns the exact entity
- [ ] `get` with an ambiguous query returns an error
- [ ] All five entity types work
- [ ] All three output formats (text, json, toml) produce valid output
