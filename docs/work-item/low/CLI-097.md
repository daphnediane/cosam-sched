# CLI-097: cosam-modify undo, redo, and show-history commands

## Summary

Implement in-memory `undo`, `redo`, and `show-history` subcommands.

## Status

Open

## Priority

Low

## Blocked By

- CLI-091: cosam-modify scaffold, file I/O, module structure

## Description

Thin wrappers around `EditContext::undo()`, `EditContext::redo()`, and the depth accessors.
Useful within a multi-stage invocation to reverse earlier stages.

```text
cosam-modify --file <path> <edit-stage> -- undo
cosam-modify --file <path> show-history
```

**Limitation:** The EditHistory undo/redo stacks are in-memory only and are NOT persisted
across invocations. CLI-099 tracks adding binary-file persistence.

### show-history output

- `text`: `undo: N` / `redo: N`
- `json`: `{"undo": N, "redo": N}`
- `toml`: `undo = N` / `redo = N`

## Acceptance Criteria

- [ ] `undo` after a `set` restores the original field value
- [ ] `redo` after `undo` re-applies the change
- [ ] `undo` with empty stack returns a clear error (not a crash)
- [ ] `show-history` reports correct counts after a series of edits and undos
