# Widget Compatibility with New Export Format

## Summary

Audit and update the calendar widget to handle the format differences between
the v9 JSON output and the format produced by `cosam-convert` (CLI-030).

## Status

Completed

## Priority

High

## Description

`cosam-convert` produces a JSON format that differs from what the v9 toolchain
produced. The widget must be audited and updated to handle these differences
before the new toolchain can be deployed.

### Differences to Address

#### 1. Version field reset to 0

`meta.version` is `0` in the new output (previously `9`). The widget must not
attempt to up-convert from version 9 or reject version 0.

#### 2. SPLIT/SPLITDAY/SPLITNIGHT rooms removed

v9 used fake rooms (`SPLIT`, `SPLITDAY`, `SPLITNIGHT`) to represent meal breaks
and day boundaries. These no longer appear in the output. Verify the widget
degrades gracefully when panels that were previously in those rooms now have
`roomIds: []` or are assigned to a real break-type room.

#### 3. BREAK room — `isBreak` not set correctly during import

The `BREAK` room currently exports with `isBreak: false` because the XLSX import
does not yet set the `isBreak` flag on `EventRoomCommonData`. Until the import
fix lands, the widget should also check the panel type's `is_break` flag to
identify break panels, not rely on `isBreak` on the room alone.

#### 4. Null / default fields omitted from panel objects

Optional panel fields (`note`, `prereq`, `cost`, `capacity`, `difficulty`,
`ticketUrl`) are now omitted when null rather than serialized as `null`. Boolean
flags (`isFree`, `isFull`, `isKids`) are omitted when `false`. Array fields
(`roomIds`, `credits`, `presenters`) are omitted when empty. The widget must
treat absent fields as their zero/null/empty equivalent.

#### 5. Schedule bounds timestamps — local time, no timezone suffix

`meta.startTime`/`endTime` are now local datetime strings (`2026-06-25T17:00:00`)
without a timezone suffix, matching the per-panel `startTime`/`endTime` fields.
Previously the meta bounds were UTC with a `Z` suffix. The widget should not
attempt timezone conversion on these values.

## Acceptance Criteria

- Widget renders correctly when loaded with a version-0 export
- Panels with `roomIds: []` are handled gracefully (shown without a room or
  omitted from the room grid, not a crash)
- Widget identifies break panels via the panel type's `is_break` flag
- Widget handles absent optional fields without errors or display glitches
- Widget does not interpret `startTime`/`endTime` values as UTC
