# CLI-139: Command-line layout jobs in cosam-convert; remove cosam-layout

## Summary

Replace the standalone `cosam-layout` binary with `--layout.<key>=<value>`
flags on `cosam-convert`.

## Status

Completed

## Priority

Medium

## Description

The standalone `cosam-layout` app duplicated `cosam-convert`'s in-process layout
pipeline (both call `schedule-layout::document::generate`) and was the only
front-end still parsing per-job `--content`/`--paper`/`--split` flags. It is
removed; `cosam-convert` now supports defining a single layout job directly on
the command line.

## Implementation Details

- Removed `apps/cosam-layout` and dropped it from the workspace members.
- Added `OutputSettings.layout: Option<JobConfig>`, populated by repeatable
  `--layout.<key>[=<value>]` flags (keys mirror `config/layout.toml` field
  names; hyphens accepted as underscores; bare boolean flags mean `true`;
  `--layout.import` stacks presets).
- When a command-line layout is set, `--export-layout <file>` renders just that
  one job, using the path as the output file name (written verbatim when it has
  an extension). Imports resolve against the presets from **both**
  `layout-default.toml` and the user's `layout.toml` (user wins on clashes).
- `--layout-config <file>`, `--default-layouts`, and `--default` discard any
  accumulated `--layout.*` and revert to rendering the layout-TOML jobs.
- `run_layout_export` now branches on the command-line layout vs. TOML jobs and
  shares one writer; per-job font args come from the job's resolved brand.

## Acceptance Criteria

- [x] `cosam-layout` removed from the workspace; `cargo build --workspace` green.
- [x] `--layout.import=flyer --layout.paper=letter --export-layout out/flyer.pdf`
      writes `out/flyer.pdf` verbatim.
- [x] `--export-layout <dir>` without `--layout.*` still renders the TOML jobs
      into per-paper subdirectories.
- [x] `--default-layouts` after `--layout.*` reverts to the TOML jobs.
- [x] `cargo test -p cosam-convert` passes.
