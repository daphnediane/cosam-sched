# UI-143: Keep the day visible while scrolling (sticky headers + modal date)

## Summary

Make the day stay visible in list and grid views while scrolling, and show the date in the event detail modal.

## Status

Completed

## Priority

Medium

## Description

In list view (especially multi-day "All Days") the day was easy to lose while
scrolling, because only the first time-group of each day carried the day label
and the sticky time headers had a transparent background that let panels show
through. Grid view had the same problem once the start of the current day
scrolled above the top of the window, and its room-name header row was not
actually sticky (it was `position: relative`). The detail modal showed the time
range but no day at all, so screenshots people share of a single panel lost
which day it belonged to.

## Implementation Details

- **Detail modal**: lead the meta row with the day (weekday + date) using a new
  calendar icon, emphasized so screenshots keep the date.
- **List view**: group events into per-day `.cosam-day-section`s with a sticky
  `.cosam-day-header` (pins for the whole day) and a sticky `.cosam-time-header`
  pinned just beneath it. Both use a solid background so panels scroll under,
  not through.
- **Grid view**: repeat a sticky header row at each day. Its time-column corner
  shows the weekday over the date (mirroring the room name / hotel room split)
  and the room columns repeat, so the day and the room labels stay visible and
  swap cleanly at a day boundary. Replaced an earlier full-width "day band"
  approach that awkwardly overlapped the room headers mid-scroll.
- Made the grid scroll inside `.cosam-grid-view` (`overflow: auto;
  max-height: 80vh`) and removed `overflow: hidden` from `.cosam-grid`, since
  the prior overflow ancestors prevented page-relative sticky from ever
  triggering. z-index layering: header row (25) above events (20) / breaks (10).

## Acceptance Criteria

- [x] Detail modal shows the day (weekday + date)
- [x] List view day header stays pinned while scrolling within a day
- [x] List view headers have an opaque background (panels do not show through)
- [x] Grid view day + room headers stay pinned and swap cleanly per day
- [x] List view background behind the Date / Time headers matches the page /
      div background in the **default theme** (headers now source the detected
      host page color; other themes keep their own opaque header color)
- [x] Grid view has no empty trailing time slot / spurious day header below the
      last event (end-only boundary keys fold onto the grid `[footer]` line)
- [x] Grid view scrolls with the page (toolbar scrolls up to the top, then the
      header row pins) instead of trapping the scroll in an inner box

## Notes

- **Default-theme header color (resolved):** the list day/time headers now use
  the `--cosam-time-header-bg` token (previously they hard-coded `--cosam-bg`,
  a near-white surface that read as white over a colored host page). In the
  transparent default theme that token resolves to
  `var(--cosam-page-bg, var(--cosam-bg))`; the widget JS
  (`_applyHeaderPageColor`) walks the widget's ancestors, finds the first
  opaque background, and sets `--cosam-page-bg` on the root. The headers stay
  opaque (panels still scroll under them) while matching the page color.
  Named themes keep their own opaque `--cosam-time-header-bg`.
- **Grid trailing slot (resolved):** time slots derive from event start *and*
  end times, so the global-max end key produced an empty bottom row (and a
  spurious next-day header). `_buildGridView` now folds such end-only boundary
  keys onto the grid `[footer]` line instead of emitting tracks for them.
- **Grid page scroll (resolved):** `.cosam-grid-view` dropped its inner
  `overflow: auto; max-height: 80vh` for `overflow: visible`, so the page
  scrolls the grid (toolbar scrolls away, header row pins via sticky `top: 0`).
  Room columns are flexible (`minmax(0, 1fr)`) so the grid fits without needing
  its own horizontal scroll.
- Minified bundles (`widget/cosam-calendar.min.*`) are gitignored and rebuild
  via `npm run build`.
